## 安全与可观察性

### 按需钩子技能（/careful, /freeze, /guard）

**内容：** 三个新技能，使用 Claude Code 的会话范围 PreToolUse 钩子按需添加安全护栏。

**原因：** Anthropic 的内部技能最佳实践推荐按需钩子用于安全。Claude Code 已经处理破坏性命令权限，但这些为高风险会话（触摸生产、调试实时系统）添加了显式选择加入层。

**技能：**
- `/careful` — Bash 工具的 PreToolUse 钩子。在破坏性命令前警告（不阻止）：`rm -rf`、`DROP TABLE`、`git push --force`、`git reset --hard`、`kubectl delete`、`docker system prune`。使用 `permissionDecision: "ask"` 以便用户可以覆盖。
- `/freeze` — Edit/Write 工具的 PreToolUse 钩子。将文件编辑限制在用户指定的目录。非常适合调试而不意外"修复"无关代码。
- `/guard` — 元技能，将 `/careful` + `/freeze` 组合成一个命令。

**实现说明：** 在钩子命令中使用 `${CLAUDE_SKILL_DIR}`（不是 `${SKILL_DIR}`）作为脚本路径。纯 bash JSON 解析（无 jq 依赖）。冻结目录存储：`${CLAUDE_PLUGIN_DATA}/freeze-dir.txt` 与 `~/.gstack/freeze-dir.txt` 回退。确保冻结目录路径上的尾随 `/` 以防止 `/src` 匹配 `/src-old`。

**工作量：** M（人工）/ S（CC）
**优先级：** P3
**依赖：** 无

### 技能使用遥测

**内容：** 跟踪哪些技能被调用，频率如何，来自哪个仓库。

**原因：** 能够发现触发不足的技能并衡量采用情况。Anthropic 使用 PreToolUse 钩子进行此操作；更简单的方法是从前言追加 JSONL。

**背景：** 添加到 `scripts/gen-skill-docs.ts` 中的 `generatePreamble()`。追加到 `~/.gstack/analytics/skill-usage.jsonl`，包含技能名称、时间戳和仓库名称。`mkdir -p` 确保目录存在。

**工作量：** S（人工）/ S（CC）
**优先级：** P3
**依赖：** 无
