# TODOS.md 格式参考

TODOS.md 规范格式的共享参考。被 `/ship`（步骤 5.5）和 `/plan-ceo-review`（TODOS.md 更新部分）引用，以确保一致的 TODO 项目结构。

---

## 文件结构

```markdown
# TODOS

## <技能/组件>     ← 例如，## 浏览，## 发布，## 审查，## 基础设施
<项目按 P0 优先排序，然后是 P1、P2、P3、P4>

## 已完成
<带有完成注释的已完成项目>
```

**部分：** 按技能或组件组织（`## 浏览`、`## 发布`、`## 审查`、`## QA`、`## 回顾`、`## 基础设施`）。在每个部分内，按优先级排序（P0 在顶部）。

---

## TODO 项目格式

每个项目是其部分下的 H3：

```markdown
### <标题>

**内容：** 工作的一行描述。

**原因：** 它解决的具体问题或解锁的价值。

**背景：** 足够的细节，使 3 个月后接手的人理解动机、当前状态和从哪里开始。

**工作量：** S / M / L / XL
**优先级：** P0 / P1 / P2 / P3 / P4
**依赖于：** <先决条件，或 "无">
```

**必需字段：** 内容、原因、背景、工作量、优先级
**可选字段：** 依赖于、被阻塞

---

## 优先级定义

- **P0** — 阻塞：必须在下次发布前完成
- **P1** — 关键：本周期应完成
- **P2** — 重要：当 P0/P1 完成后再做
- **P3** — 锦上添花：在采用/使用数据后重新审视
- **P4** — 有朝一日：好主意，无紧迫感

---

## 已完成项目格式

当项目完成时，将其移至 `## 已完成` 部分，保留其原始内容并附加：

```markdown
**已完成：** vX.Y.Z (YYYY-MM-DD)
```
