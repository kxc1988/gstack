# 设计审查清单（精简版）

> **DESIGN_METHODOLOGY 的子集** — 在此处添加项目时，请同时更新 `scripts/gen-skill-docs.ts` 中的 `generateDesignMethodology()`，反之亦然。

## 说明

此清单适用于**差异中的源代码** — 而非渲染输出。阅读每个更改的前端文件（完整文件，而非仅差异块）并标记反模式。

**触发条件：** 仅当差异触及前端文件时运行此清单。使用 `gstack-diff-scope` 检测：

```bash
eval $(~/.claude/skills/gstack/bin/gstack-diff-scope <base> 2>/dev/null)
```

如果 `SCOPE_FRONTEND=false`，则静默跳过整个设计审查。

**DESIGN.md 校准：** 如果仓库根目录中存在 `DESIGN.md` 或 `design-system.md`，请先阅读它。所有发现都根据项目的既定设计系统进行校准。在 DESIGN.md 中明确认可的模式不会被标记。如果不存在 DESIGN.md，则使用通用设计原则。

---

## 置信度等级

每个项目都标记有检测置信度级别：

- **[高]** — 通过 grep/模式匹配可靠检测。明确的发现。
- **[中]** — 通过模式聚合或启发式检测。标记为发现但预期会有一些噪音。
- **[低]** — 需要理解视觉意图。呈现为："可能的问题 — 请通过视觉验证或运行 /design-review。"

---

## 分类

**自动修复**（仅机械 CSS 修复 — 高置信度，不需要设计判断）：
- `outline: none` 没有替代方案 → 添加 `outline: revert` 或 `&:focus-visible { outline: 2px solid currentColor; }`
- 新 CSS 中的 `!important` → 删除并修复特异性
- 正文字体 `font-size` < 16px → 增加到 16px

**需要输入**（其他所有内容 — 需要设计判断）：
- 所有 AI 草率发现、排版结构、间距选择、交互状态差距、DESIGN.md 违规

**低置信度项目** → 呈现为 "可能：[描述]。请通过视觉验证或运行 /design-review。" 永远不要自动修复。

---

## 输出格式

```
设计审查：N 个问题（X 个可自动修复，Y 个需要输入，Z 个可能）

**已自动修复：**
- [file:line] 问题 → 已应用修复

**需要输入：**
- [file:line] 问题描述
  推荐修复：建议的修复方案

**可能（请通过视觉验证）：**
- [file:line] 可能的问题 — 请使用 /design-review 验证
```

如果未发现问题：`设计审查：未发现问题。`

如果没有更改前端文件：静默跳过，无输出。

## 类别

### 1. AI 草率检测（6 项）— 最高优先级

这些是 AI 生成 UI 的明显迹象，受尊敬的工作室的设计师不会发布这些。

- **[中]** 紫色/紫罗兰/靛蓝色渐变背景或蓝到紫的配色方案。查找 `linear-gradient` 中值在 `#6366f1`–`#8b5cf6` 范围内的，或解析为紫色/紫罗兰色的 CSS 自定义属性。

- **[低]** 3 列功能网格：彩色圆圈中的图标 + 粗体标题 + 2 行描述，对称重复 3 次。查找具有恰好 3 个子元素的网格/弹性容器，每个子元素都包含一个圆形元素 + 标题 + 段落。

- **[低]** 彩色圆圈中的图标作为章节装饰。查找具有 `border-radius: 50%` + 背景颜色的元素，用作图标的装饰容器。

- **[高]** 所有内容居中：所有标题、描述和卡片上的 `text-align: center`。搜索 `text-align: center` 的密度 — 如果 >60% 的文本容器使用居中对齐，则标记它。

- **[中]** 每个元素上统一的气泡状边框半径：相同的大半径（16px+）统一应用于卡片、按钮、输入、容器。汇总 `border-radius` 值 — 如果 >80% 使用相同的值 ≥16px，则标记它。

- **[中]** 通用英雄文案："欢迎来到 [X]"、"释放...的力量"、"您的一站式解决方案..."、"彻底改变您的..."、"简化您的工作流程"。在 HTML/JSX 内容中搜索这些模式。

### 2. 排版（4 项）

- **[高]** 正文字体 `font-size` < 16px。搜索 `body`、`p`、`.text` 或基础样式上的 `font-size` 声明。值低于 16px（或当基础为 16px 时的 1rem）会被标记。

- **[高]** 差异中引入超过 3 种字体系列。计算不同的 `font-family` 声明。如果在更改的文件中出现 >3 个唯一系列，则标记。

- **[高]** 标题层次结构跳过级别：`h1` 后跟 `h3`，同一文件/组件中没有 `h2`。检查 HTML/JSX 中的标题标签。

- **[高]** 黑名单字体：Papyrus、Comic Sans、Lobster、Impact、Jokerman。搜索 `font-family` 中的这些名称。

### 3. 间距和布局（4 项）

- **[中]** 当 DESIGN.md 指定间距比例时，不在 4px 或 8px 比例上的任意间距值。将 `margin`、`padding`、`gap` 值与指定的比例进行比较。仅当 DESIGN.md 定义了比例时才标记。

- **[中]** 没有响应式处理的固定宽度：容器上的 `width: NNNpx`，没有 `max-width` 或 `@media` 断点。在移动设备上有水平滚动的风险。

- **[中]** 文本容器上缺少 `max-width`：正文文本或段落容器没有设置 `max-width`，允许行超过 75 个字符。检查文本包装器上的 `max-width`。

- **[高]** 新 CSS 规则中的 `!important`。在添加的行中搜索 `!important`。几乎总是应该正确修复的特异性逃生舱口。

### 4. 交互状态（3 项）

- **[中]** 交互元素（按钮、链接、输入）缺少悬停/焦点状态。检查新交互元素样式是否存在 `:hover` 和 `:focus` 或 `:focus-visible` 伪类。

- **[高]** `outline: none` 或 `outline: 0` 没有替代焦点指示器。搜索 `outline:\s*none` 或 `outline:\s*0`。这会移除键盘可访问性。

- **[低]** 交互元素上的触摸目标 < 44px。检查按钮和链接上的 `min-height`/`min-width`/`padding`。需要从多个属性计算有效大小 — 仅从代码中获得低置信度。

### 5. DESIGN.md 违规（3 项，有条件）

仅当存在 `DESIGN.md` 或 `design-system.md` 时应用：

- **[中]** 不在既定调色板中的颜色。将更改的 CSS 中的颜色值与 DESIGN.md 中定义的调色板进行比较。

- **[中]** 不在既定排版部分中的字体。将 `font-family` 值与 DESIGN.md 的字体列表进行比较。

- **[中]** 超出既定比例的间距值。将 `margin`/`padding`/`gap` 值与 DESIGN.md 的间距比例进行比较。

## 抑制

不要标记：
- 在 DESIGN.md 中明确记录为有意选择的模式
- 第三方/供应商 CSS 文件（node_modules、vendor 目录）
- CSS 重置或标准化样式表
- 测试夹具文件
- 生成/压缩的 CSS
