# gstack 翻译计划

## 项目概述

gstack 是一个由 Y Combinator CEO Garry Tan 开发的开源软件工厂，它为 Claude Code 提供了一套结构化的工作流技能，将其转变为一个虚拟工程团队。项目包含多个核心文档，需要从英文翻译成中文，以便中文用户更好地理解和使用。

## 翻译目标

- 将 gstack 的核心文档翻译成中文，保持原意的同时确保语言流畅自然
- 建立统一的翻译标准和术语表，确保翻译的一致性
- 保留技术术语的准确性，同时使非技术用户也能理解
- 确保翻译后的文档格式与原文一致，包括代码块、表格等

## 翻译范围

### 核心文件（优先级 P0）

1. **README.md** - 项目的主要说明文档，包含项目介绍、安装指南、使用示例等
2. **ARCHITECTURE.md** - 架构设计文档，解释系统设计原理和内部结构
3. **BROWSER.md** - 浏览器工具的技术细节和命令参考
4. **docs/skills.md** - 所有技能的详细说明和使用指南

### 技能文件（优先级 P1）

1. **browse/SKILL.md** - 浏览器技能的详细说明
2. **review/SKILL.md** - 代码审查技能的详细说明
3. **qa/SKILL.md** - QA 测试技能的详细说明
4. **ship/SKILL.md** - 发布技能的详细说明
5. **office-hours/SKILL.md** - 办公时间技能的详细说明

### 其他文件（优先级 P2）

1. **CONTRIBUTING.md** - 贡献指南
2. **CHANGELOG.md** - 版本变更记录
3. **CLAUDE.md** - Claude 配置文件
4. **TODOS.md** - 待办事项列表

## 翻译标准

1. **术语一致性**：建立统一的术语表，确保技术术语在整个项目中的翻译一致
2. **准确性**：保持原文的技术准确性，特别是代码、命令和架构描述
3. **流畅性**：确保翻译后的中文自然流畅，符合中文表达习惯
4. **格式保持**：保持原文的格式，包括标题层级、代码块、表格等
5. **完整性**：确保所有内容都被翻译，不遗漏任何部分

## 翻译流程

1. **准备阶段**
   - 建立术语表，确定关键术语的中文翻译
   - 分析原文结构，确定翻译的优先级和顺序
   - 熟悉项目的技术背景和上下文

2. **翻译阶段**
   - 按照优先级顺序翻译核心文件
   - 保持翻译的一致性和准确性
   - 定期检查翻译质量，确保语言流畅自然

3. **校对阶段**
   - 对翻译后的文档进行校对，检查是否有遗漏或错误
   - 确保技术术语的准确性和一致性
   - 检查格式是否与原文一致

4. **交付阶段**
   - 整理翻译后的文档，确保所有文件都已翻译完成
   - 提供翻译总结和术语表
   - 确保翻译后的文档可以正常显示和使用

## 翻译文件保存方式

**重要说明**：我们不会替换原文文件，而是会生成新的翻译文件，与原文文件并存。

- 翻译后的文件将保存在与原文相同的目录中，添加 `.zh-CN` 后缀
- 例如：`README.md` 的翻译文件为 `README.zh-CN.md`
- 对于技能文件，保持相同的命名规范
- 这样可以确保原文和翻译版本都能被访问，方便不同语言的用户使用

2. **语言流畅性**：确保翻译后的中文自然流畅，符合中文表达习惯
3. **格式一致性**：确保翻译后的文档格式与原文一致
4. **内容完整性**：确保所有内容都被翻译，不遗漏任何部分

## 风险与应对

2. **上下文理解**：深入了解项目背景，确保翻译的准确性
3. **时间管理**：合理安排时间，确保按时完成翻译任务
4. **质量控制**：定期检查翻译质量，确保翻译的准确性和流畅性
